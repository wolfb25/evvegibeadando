A következő funkciókat találjátok:

Menü [ F2 ]: A common_studio13.js / menü_json változó tartalma: A mezőnevek értékeinek értelemszerű átírásával beállítható a kívánt menü; tip=0 esetén id="main1" TAG-be kerül az URL értéke (ennek a szövegnek a helyére); tip=1: url oldal betöltése a jelenlegi oldal helyett; tip=2: u.a. mint egy, csak, új lapra töltődik be az oldal.

Gyorskereső mező [ F4 ]: Keresésnek fenntartott hely, több táblára, összesítve, kategorizálva. A  search1_get() eljárás a helye ennek az algoritmusnak.

Bejelentkezés / kijelentkezés [ F8 ]: Az id="login1_modal" modális ablakot jeleníti meg. A bejelentkezett user nevét célszerű kitenni az id="login1_user" TAG-be.

Üzenetablak: Adott színű szöveges üzenetet jelenít meg a jobb alsó sarokban. a tip értéke lehet "success", "info", "warning" és "error" (változó színekkel). Ha az mp nem nulla, akkor adott másodperc után eltűnik az üzenet.

 mySend({text: `Az adatok sikeresen elmentve.`, tip:"info", mp:3});

myQuestion({text: `Pite ...? `});

Backend → datamodul_mysql.js:

A minta keretrendszer a create_IT_termekek.sql adatbázishoz kapcsolódik.

"Pool" technikát használó connection.JSON formátumot ad vissza. A paraméterként kapott SQL stringről eldönti, hogy adatkarbantartó (INSERT UPDATE, DELETE), vagy SELECT utasítás. Select utasításnál megvizsgálja a max elemszámot, valamint a limit és offset értékét.  (lásd: "JSON Div tábla" menü).

A SELECT JSON formátuma:

text: "az esetleges mysql hibaüzenet". Ha minden ok, akkor "0"

tip: van találat? "info": "warning" → jó lesz a "mysend" üzenet ablakhoz.

maxcount: limit nélkül ennyi a rekordszám

limit: az SQL selectben beállított limit.

offset: az SQL selectben beállított offset.

pages: A select-et az akt. limit értékkel ennyi lapra lehet felbontani
(pl: 1000 találat -maxcount- limit 20 = 50 lap)

count: limit rekordszám (<= limit)

rows: a select rekordjai [tömb]

Az INSERT, UPDATE, DELETE JSON formátuma:

text: "az esetleges mysql hibaüzenet". Ha minden ok, akkor "adatkarbantartás OK…" az üzenet.

tip: van találat? "info": "warning" , vagy "error"

count: a karbantartott rekordok száma. (A "0" hibát jelezhet…)

Napló:

Minden (SQL) tevékenység naplózásra kerül. Tárolódik a felhasználónév, IP cím, SQL és az időbélyeg.

Login / logout: 

Backend Session - Frontend Süti páros tárolja a bejelentkezett user adatait.
